# ボートレース AI 自動予測システム 仕様書

**バージョン:** 1.0  
**作成日:** 2026-04-01  
**目的:** 実際の投資運用を目的としたボートレース予測システムの構築

---

## 1. システム概要

ボートレースの過去データ・当日情報を収集し、LightGBMベースの機械学習モデルで各艇の着順確率を予測する。予測結果とオッズから期待値を算出し、投資判断を支援する。

### 1.1 最終的なユーザーフロー

```
[毎朝] 出走表データ取得
    ↓
[レース2時間前] 直前情報（展示タイム・オッズ）取得
    ↓
[予測実行] 各艇の1着確率・期待値を表示
    ↓
[レース10分前] スタート展示を取得
    ↓
[予測実行] 各艇の1着確率・期待値を表示
    ↓
[手動投票] ユーザーが舟券を購入
    ↓
[レース後] 結果取得・収益記録・モデル更新用データ蓄積
```

## 2. 機能要件

### 2.1 データ収集機能

#### 2.1.1 過去データ収集（バッチ・初回のみ）


| データ種別 | ソース               | 取得内容             |
| ----- | ----------------- | ---------------- |
| レース結果 | Boatrace Open API | 着順・配当・STタイム・決まり手 |
| 番組表   | 公式ダウンロードデータ       | 出走選手・コース・モーター番号  |
| 選手成績  | 公式ダウンロードデータ       | 勝率・2連率・3連率・期別成績  |


- 取得期間：直近3年分（2023年〜現在）を優先
- ストレージ：SQLiteデータベースに格納

#### 2.1.2 当日データ収集（スケジュール実行）


| データ種別           | ソース                 | 取得タイミング    | 手法                    |
| --------------- | ------------------- | ---------- | --------------------- |
| 出走表             | boatrace.jp スクレイピング | 前日〜当日 8:00 | metaboatrace/scrapers |
| 直前情報            | boatrace.jp スクレイピング | レース2時間前    | Playwright            |
| オッズ（単勝・2連単・3連単） | boatrace.jp スクレイピング | レース直前      | Playwright            |
| 気象データ           | Open-Meteo API      | レース直前      | REST API              |


- スクレイピング間隔：最低3秒以上のwaitを設ける
- 失敗時はリトライ（最大3回）後スキップしてログ記録

#### 2.1.3 取得データ項目詳細

**選手・モーター関連**

- 選手登録番号・名前・年齢・支部・体重・級別（A1/A2/B1/B2）
- 全国勝率・全国2連率・当地勝率・当地2連率
- モーター番号・モーター2連率・ボート番号・ボート2連率
- 直近の成績（過去6走）

**レース環境**

- 競艇場コード・レース番号
- 天気・気温・水温・風向・風速・波高
- コース別スタート展示タイム

**オッズ**

- 単勝（1着）オッズ
- 2連単・2連複オッズ
- 3連単・3連複オッズ（メイン予測対象）

### 2.2 予測機能

#### 2.2.1 予測モデル

**メインモデル：LightGBM（多クラス分類）**

- 予測対象：各艇の1着確率（6艇分のソフトマックス出力）
- 入力特徴量：約50〜80次元（選手成績・モーター成績・コース・展示タイム・気象等）
- 学習データ：過去3年分のレース結果
- 評価指標：LogLoss・的中率・回収率

**補助モデル（第2フェーズ以降）**

- 2着・3着確率モデルを追加し3連単予測に拡張

#### 2.2.2 特徴量一覧（主要）


| カテゴリ  | 特徴量                       |
| ----- | ------------------------- |
| 選手実力  | 全国勝率・2連率、当地勝率・2連率、級別(数値化) |
| コース適性 | コース別1着率（1〜6コース各々）         |
| 調子    | 直近6走の平均着順・1着回数            |
| モーター  | モーター2連率、使用節数              |
| スタート  | 展示STタイム、平均フライング率          |
| レース環境 | 風速・風向（cos/sin変換）・水温・波高    |
| コース   | インコース（1コース）フラグ、枠番         |
| オッズ   | 単勝オッズ（市場評価として特徴量に含める）     |


#### 2.2.3 期待値計算

```
期待値 = 予測1着確率 × 単勝オッズ

期待値 > 1.0 → プラス期待値（投資候補）
期待値 > 1.2 → 推奨
```

3連単の場合：

```
3連単期待値 = P(1着) × P(2着|1着) × P(3着|1,2着) × 3連単オッズ
```

### 2.3 表示・UI機能

**Streamlitによるウェブダッシュボード**

#### 予測画面

- 競艇場・レース番号を選択

- 日別・月別 投資額 / 回収額 / 回収率

#### データ確認画面

- 最新の取得状況（最終更新日時）
- 選手・モーター成績の検索

## 5. データベース設計（主要テーブル）

### races（レース基本情報）


| カラム               | 型       | 説明                  |
| ----------------- | ------- | ------------------- |
| race_id           | TEXT PK | `{場コード}{日付}{レース番号}` |
| stadium_code      | TEXT    | 競艇場コード（01〜24）       |
| race_date         | DATE    | 開催日                 |
| race_number       | INT     | レース番号（1〜12）         |
| weather           | TEXT    | 天気                  |
| temperature       | REAL    | 気温                  |
| water_temperature | REAL    | 水温                  |
| wind_speed        | REAL    | 風速                  |
| wind_direction    | INT     | 風向                  |
| wave_height       | INT     | 波高                  |


### race_entries（出走情報）


| カラム             | 型          | 説明      |
| --------------- | ---------- | ------- |
| entry_id        | INTEGER PK |         |
| race_id         | TEXT FK    |         |
| boat_number     | INT        | 艇番（1〜6） |

| カラム                  | 型       | 説明     |
| -------------------- | ------- | ------ |
| racer_id             | TEXT PK | 選手登録番号 |

### race_results（レース結果）


| カラム          | 型          | 説明             |
| ------------ | ---------- | -------------- |
| result_id    | INTEGER PK |                |

| boat_number  | INT        | 艇番             |
| arrival      | INT        | 着順             |

| カラム         | 型          | 説明                                         |

## 8. リスクと注意事項


| リスク                           | 対策                            |
| ----------------------------- | ----------------------------- |
| boatrace.jp の仕様変更によるスクレイピング失敗 | エラー通知 + Open APIをフォールバックとして利用 |
| モデルの過学習                       | 時系列分割による適切なバックテスト             |
| 賭けすぎによる損失                     | 期待値1.2以上のみ・1レース投資額の上限設定       |
| データの取得漏れ                      | 取得ログを確認し、欠損レースを検知する仕組み        |

## 9. 免責事項

本システムは予測補助ツールであり、投資利益を保証するものではない。  
公営ギャンブルのため18歳以上のみ利用可能。資金管理は自己責任で行うこと。
